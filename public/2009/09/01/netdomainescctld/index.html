Net::Domain::ES::ccTLD

Tue 01 Sep 2009 03:28:23 PM UTC

As I’m rewriting the core of Planeta Linux (you can track progress here), I needed a reliable way to map a country TLD to its Spanish name. I headed to metaCPAN to see what was already built to do that and found out Locales::Country that looked awesome but didn’t work for me, code2country('ni') would return undef for me, plus the countries that were actually there had the encoding all fubar, so fuck it, drop it.

So I came up with Net::Domain::ES::ccTLD using the data from the Spanish Wikipedia for the countries TLDs:

use Net::Domain::ES::ccTLD;

my $country = find_name_by_cctld('mx')     # $country is 'MÃ©xico'
  or die "Couldn't find name.";

my $current = find_name_by_cctld('us');    # $current is 'Estados Unidos';

Why would I reinvent the wheel for this? Because it was easier and faster for me and it’s all about practicality, isn’t it? Plus the code is super tiny, maybe the smallest module I have in CPAN in terms of actual code.

Feel free to use if you will.
